## Context

当前扩展使用基础的 CSS 样式，配色为 #1a1a2e 蓝黑色背景和 #e94560 红色强调色，按钮有明显边框，编辑态有双边框问题。需要全面重构为 Google Gemini / Material Design 3 风格。

## Goals / Non-Goals

**Goals:**
- 实现 Material Design 3 深色主题视觉系统
- 按钮扁平化 + 交互动效
- 编辑态无边框融合设计
- 药丸形搜索框
- 全局平滑过渡动画

**Non-Goals:**
- 不改变功能逻辑
- 不引入外部 CSS 框架或图标库
- 不改变整体布局结构（仅美化）

## Decisions

### 1. 配色方案

| 角色 | 颜色值 |
|---|---|
| 主背景 | #1e1f20 |
| 表面/卡片 | #282a2c |
| 表面高亮 | #333537 |
| 边框/分割线 | rgba(255,255,255,0.1) |
| 强调色 | #8ab4f8（Gemini 浅蓝） |
| 强调色变体 | #6ea6f0（悬停） |
| 文本主色 | #e3e3e3 |
| 文本次色 | #9aa0a6 |
| 删除色 | #f28b82 |
| 删除背景 | rgba(242,139,130,0.12) |

### 2. 圆角策略

- 容器/卡片/弹窗：12px
- 按钮/输入框：10px
- 搜索框（药丸形）：999px
- 编辑态输入框：6px（贴合单元格）

### 3. 编辑态设计

**选择**：无边框透明设计，仅保留底部 2px 强调色线条，背景使用极淡的 rgba(138,180,248,0.06)。

**理由**：消除双边框问题的关键是不在 td.editing 上加 box-shadow 的同时又在 input 上加 border。改为只用底边框 + 无 outline。

### 4. 搜索图标方案

**选择**：使用 SVG 内联图标（放大镜），通过 CSS 的 `background-image` + `padding-left` 实现。

**理由**：无需引入图标库，保持零依赖。

## Risks / Trade-offs

- [SVG 兼容性] Chrome 扩展内 SVG 渲染无问题 → 无风险。
- [动画性能] transition 动画在低端设备可能卡顿 → 缓解：仅对 transform 和 opacity 做动画，避免 layout 属性。
