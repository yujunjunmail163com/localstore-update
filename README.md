# Local Storage Manager

一个 Chrome 浏览器扩展，用于直接查看、编辑和管理当前网页的 `localStorage` 数据。支持行内编辑（Key / Value）、搜索过滤、新增和删除，采用 Material Design 3 深色主题。

---

## 功能特性

- **查看 localStorage**：一键读取当前标签页域名下的所有 localStorage 数据，以表格形式展示。
- **行内编辑 Key**：点击 Key 列单元格即可直接修改 Key 名称（自动完成 delete + set 重命名）。
- **行内编辑 Value**：点击 Value 列单元格即可直接修改 Value 值，支持多行文本自动切换为 textarea。
- **新增数据**：通过弹窗表单添加新的 Key-Value 数据。
- **删除数据**：删除单条数据（带二次确认弹窗）。
- **搜索过滤**：药丸形搜索框，实时按 Key 名称过滤数据。
- **刷新数据**：手动刷新当前标签页的 localStorage 数据。
- **Material Design 3 深色主题**：现代化的 UI 设计，扁平化按钮、平滑过渡动画、大圆角布局。

## 安装步骤

### 方式一：开发者模式加载（推荐）

1. **下载项目代码**

   将本项目克隆或下载到本地任意目录：
   ```
   git clone <仓库地址>
   ```
   或直接下载 ZIP 文件并解压。

2. **打开 Chrome 扩展管理页面**

   在 Chrome 浏览器地址栏中输入以下地址并回车：
   ```
   chrome://extensions/
   ```

3. **启用开发者模式**

   在扩展管理页面的 **右上角**，找到「开发者模式」开关，将其打开（切换为蓝色/开启状态）。

4. **加载扩展**

   点击页面左上角出现的 **「加载已解压的扩展程序」** 按钮，在弹出的文件选择器中，选择本项目的 **根目录**（即包含 `manifest.json` 的文件夹），点击「选择」。

5. **确认加载成功**

   扩展列表中出现 **Local Storage Manager** 卡片，且没有红色错误提示，即表示加载成功。

## 使用方法

### 第一步：打开扩展面板

在浏览器右上角的工具栏中，点击 **拼图图标（扩展程序）**，在弹出列表中找到并点击 **Local Storage Manager**。扩展面板会以弹出窗口的形式显示。

> 如果工具栏中未显示该图标，点击拼图图标后，找到 Local Storage Manager 旁边的 **固定图标（📌）** 将其固定到工具栏。

### 第二步：查看数据

打开面板后，扩展会 **自动加载** 当前标签页域名下的所有 localStorage 数据，以表格形式展示，包含以下列：

| 列名 | 说明 |
|---|---|
| # | 数据序号 |
| Key | localStorage 的键名 |
| Value | localStorage 的值 |
| 操作 | 删除按钮 |

如果当前页面没有 localStorage 数据，会显示「暂无 localStorage 数据」的提示。

### 第三步：搜索数据

在页面右上角的搜索框中输入关键字，表格会 **实时过滤**，只显示 Key 名称包含该关键字的数据。搜索框支持模糊匹配，不区分大小写。

### 第四步：新增数据

1. 点击右上角的 **「+ 新增」** 按钮。
2. 在弹出的表单中，填写 **Key**（键名）和 **Value**（值）。
3. 点击 **「确定」** 保存，或点击 **「取消」** 放弃。
4. 保存后表格会自动刷新，新数据立即生效。
5. 如果 Key 名称已存在，表单会阻止重复添加。

### 第五步：编辑 Key

1. 在表格中，**单击** 任意一行的 Key 列单元格。
2. Key 文字会变为可编辑的输入框，底部显示蓝色线条，光标颜色为蓝色强调色。
3. 直接修改 Key 名称。
4. 按 **Enter** 键、**Tab** 键 或 **点击其他区域**（失焦）保存修改。
5. 按 **Escape** 键取消修改，恢复原始 Key。
6. 如果新 Key 已存在，会弹出重名提示并恢复原始名称；如果留空，也会恢复原始名称。

> **注意**：修改 Key 实际上是「删除旧 Key + 创建新 Key + 复制原 Value」的操作。

### 第六步：编辑 Value

1. 在表格中，**单击** 任意一行的 Value 列单元格。
2. Value 文字会变为可编辑的输入框（短文本）或 textarea（含换行符的多行文本）。
3. 直接修改 Value 内容。
4. 按 **Enter** 键（单行）或 **Ctrl + Enter**（多行 textarea）或 **点击其他区域** 保存修改。
5. 按 **Escape** 键取消修改，恢复原始 Value。

### 第七步：删除数据

1. 在表格中，点击任意一行的 **「删除」** 按钮。
2. 弹出确认对话框，显示要删除的 Key 名称。
3. 点击 **「删除」** 确认，或点击 **「取消」** 放弃。
4. 删除后表格自动刷新。

### 第八步：刷新数据

点击右上角的 **「刷新」** 按钮，重新读取当前标签页的 localStorage 数据。适用于在网页中通过 JavaScript 修改了 localStorage 后，手动同步最新数据。

## 项目结构

```
localstore-update/
├── manifest.json            # Chrome 扩展清单文件（Manifest V3）
├── src/
│   ├── popup.html           # 弹出面板的 HTML 结构
│   ├── styles.css           # Material Design 3 深色主题样式
│   ├── storage-manager.js   # localStorage 数据操作层（通过 chrome.scripting 注入到当前标签页）
│   ├── inline-editor.js     # 行内编辑模块（Key/Value 点击编辑、保存、取消）
│   └── popup.js             # 主控制器（事件绑定、渲染、搜索、新增、删除）
└── openspec/                # OpenSpec 规范文件（开发过程记录）
```

### 技术说明

- **Chrome Extension Manifest V3**：使用 `chrome.scripting.executeScript` 将 localStorage 操作注入到当前活跃标签页中执行，确保操作的是目标页面的数据而非扩展自身数据。
- **权限要求**：`activeTab` + `scripting`，以及 `host_permissions: ["<all_urls>"]`（用于访问任意网页的 localStorage）。
- **零依赖**：纯原生 JavaScript (ES6+)、HTML5、CSS3，无任何外部框架或库。
- **事件委托**：表格使用事件委托处理点击事件，减少内存开销。
- **编辑互斥**：同一时间只允许一个单元格处于编辑状态。

## 常见问题

**Q：打开面板后显示「无法访问当前页面的 localStorage」？**

A：部分特殊的 Chrome 页面（如 `chrome://`、`chrome-extension://`、Chrome 应用商店等）不允许扩展注入脚本。请在普通网页（如 `https://` 开头的页面）上使用。

**Q：修改 Key 后数据丢失了？**

A：修改 Key 是「删除旧 Key → 创建新 Key」的非原子操作。正常情况下不会有问题，但如果在操作瞬间页面崩溃，理论上可能丢失。建议对重要数据先备份。

**Q：刷新扩展后需要重新打开面板吗？**

A：是的。在 `chrome://extensions/` 中点击刷新按钮后，需要重新点击工具栏的扩展图标打开面板。
