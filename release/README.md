# Local Storage Manager

一个 Chrome 浏览器扩展，用于直接查看、编辑和管理当前网页的 `localStorage` 数据。支持行内编辑（Key / Value）、搜索过滤、快速添加、白名单初始化、环境快速切换、快速跳转导航等功能，采用白色商务风格主题。

---

## 功能特性

- **查看 localStorage**：一键读取当前标签页域名下的所有 localStorage 数据，以表格形式展示。
- **行内编辑 Key**：点击 Key 列单元格即可直接修改 Key 名称（自动完成 delete + set 重命名）。
- **行内编辑 Value**：点击 Value 列单元格即可直接修改 Value 值，支持多行文本自动切换为 textarea。
- **快速添加**：页面顶部内置快捷添加区域，填写 Key 和 Value 后点击「添加」或按 Enter 直接写入。
- **删除数据**：删除单条数据（带二次确认弹窗）。
- **搜索过滤**：搜索框实时按 Key 名称过滤数据。
- **初始化本地开发环境**：一键为预定义的白名单 Key 写入默认值，快速搭建本地开发配置。
- **环境快速切换**：白名单 Key 可在本地 / 线上环境之间一键切换（通过 Key 重命名实现）。
- **快速跳转**：一键打开常用开发环境链接，支持动态配置。
- **智能域名适配**：根据当前页面域名自动切换界面显示（eclincloud 域名显示完整功能，其他域名显示精简模式）。

## 安装步骤

### 方式：开发者模式加载（推荐）

1. **下载项目代码**

   将本项目克隆或下载到本地任意目录：
   ```
   git clone <仓库地址>
   ```
   或直接下载 ZIP 文件并解压。

2. **打开 Chrome 扩展管理页面**

   在 Chrome 浏览器地址栏中输入以下地址并回车：
   ```
   chrome://extensions/
   ```

3. **启用开发者模式**

   在扩展管理页面的 **右上角**，找到「开发者模式」开关，将其打开（切换为蓝色/开启状态）。

4. **加载扩展**

   点击页面左上角出现的 **「加载已解压的扩展程序」** 按钮，在弹出的文件选择器中，选择本项目的 **根目录**（即包含 `manifest.json` 的文件夹），点击「选择」。

5. **确认加载成功**

   扩展列表中出现 **Local Storage Manager** 卡片，且没有红色错误提示，即表示加载成功。

## 页面布局

页面从上到下分为三个区块：

### 1. 顶部信息栏（Header）

| 区域 | 内容 |
|---|---|
| 左侧 | Logo 图标 + 应用名称（eclincloud 域名显示 "Eclin localstorage manager"，其他域名显示 "Localstorage Manager"） |

## 使用方法

### 第一步：打开扩展面板

在浏览器右上角的工具栏中，点击 **拼图图标（扩展程序）**，在弹出列表中找到并点击 **Local Storage Manager**。扩展面板会以弹出窗口的形式显示。

> 如果工具栏中未显示该图标，点击拼图图标后，找到 Local Storage Manager 旁边的 **固定图标（📌）** 将其固定到工具栏。

### 第二步：查看数据

打开面板后，扩展会 **自动加载** 当前标签页域名下的所有 localStorage 数据，以表格形式展示，包含以下列：

| 列名 | 说明 |
|---|---|
| # | 数据序号 |
| Key | localStorage 的键名 |
| Value | localStorage 的值 |
| 操作 | 环境切换按钮（白名单 Key）+ 删除按钮 |

数据排序规则：白名单 Key 排在最上方（按白名单定义顺序），新添加的 Key 排在最前。

如果当前页面没有 localStorage 数据，会显示「暂无 localStorage 数据」的提示。

### 第三步：快速添加数据

1. 在页面顶部的快速添加区域，填写 **Key** 和 **Value**。
2. 点击 **「添加」** 按钮，或在输入框中按 **Enter** 键。
3. 新添加的数据会自动出现在列表最上方。
4. Key 名称已存在时会阻止重复添加。

### 第四步：搜索数据

在搜索框中输入关键字，表格会 **实时过滤**，只显示 Key 名称包含该关键字的数据。搜索不区分大小写。

### 第五步：编辑 Key

1. **单击** 任意一行的 Key 列单元格，Key 文字会变为可编辑的输入框。
2. 按 **Enter** 键、**Tab** 键 或 **点击其他区域** 保存修改。
3. 按 **Escape** 键取消修改，恢复原始 Key。
4. 如果新 Key 已存在，会弹出重名提示并恢复原始名称。

> **注意**：修改 Key 实际上是「删除旧 Key + 创建新 Key + 复制原 Value」的操作。

### 第六步：编辑 Value

1. **单击** 任意一行的 Value 列单元格，Value 文字会变为可编辑的输入框或 textarea（多行文本）。
2. 按 **Enter** 键（单行）或 **Ctrl + Enter**（多行 textarea）或 **点击其他区域** 保存修改。
3. 按 **Escape** 键取消修改，恢复原始 Value。

### 第七步：初始化本地开发环境

1. 点击工具栏中的 **「初始化」** 按钮。
2. 系统会自动检查白名单中的 17 个 Key，为缺失的 Key 写入默认值（默认 URL 为 `http://localhost:9528`，`PASS_AUTH` 默认值为 `1`）。
3. 如果补充了缺失的 Key，会弹出完成提示；如果所有 Key 已存在，则静默刷新列表。

### 第八步：环境快速切换

1. 白名单 Key 行的操作列会显示一个 **切换图标**。
2. 点击该图标即可在本地 / 线上环境之间切换：
   - **本地 → 线上**：将 Key（如 `EDCURL`）重命名为末尾带 `1` 的形式（`EDCURL1`）。
   - **线上 → 本地**：将末尾带 `1` 的 Key（`EDCURL1`）恢复为原始名称（`EDCURL`）。
3. 切换后的 Key 依然保持白名单排序顺序。

### 第九步：快速跳转

1. 点击工具栏中的 **「快速跳转」** 按钮。
2. 弹窗中显示预配置的链接列表，点击链接会在新标签页中打开对应地址。

### 第十步：删除数据

1. 点击任意一行操作列的 **删除图标**。
2. 弹出确认对话框，确认后删除该条数据。

## 项目结构

```
localstore-update/
├── manifest.json              # Chrome 扩展清单文件（Manifest V3）
├── src/
│   ├── popup.html             # 弹出面板 HTML 结构（三区块布局）
│   ├── styles.css             # 白色商务风格样式
│   ├── iconfont.css           # iconfont 字体图标本地化样式
│   ├── logo.png               # 扩展图标
│   ├── storage-manager.js     # localStorage 数据操作层（通过 chrome.scripting 注入到当前标签页）
│   ├── inline-editor.js       # 行内编辑模块（Key/Value 点击编辑、保存、取消）
│   └── popup.js               # 主控制器（事件绑定、渲染、搜索、初始化、切换、跳转）
└── openspec/                  # OpenSpec 规范文件（开发过程记录）
```

### 技术说明

- **Chrome Extension Manifest V3**：使用 `chrome.scripting.executeScript` 将 localStorage 操作注入到当前活跃标签页中执行，确保操作的是目标页面的数据而非扩展自身数据。
- **权限要求**：`activeTab` + `scripting`，以及 `host_permissions: ["<all_urls>"]`。
- **CSP 配置**：`script-src 'self'; font-src 'self' https://at.alicdn.com;`，支持本地化 iconfont 字体图标。
- **零依赖**：纯原生 JavaScript (ES6+)、HTML5、CSS3，无任何外部框架或库。
- **智能适配**：根据页面域名自动切换界面模式（eclincloud 域名显示完整功能，其他域名隐藏初始化、快速跳转等内部功能按钮）。

## 常见问题

**Q：打开面板后显示「无法访问当前页面的 localStorage」？**

A：部分特殊的 Chrome 页面（如 `chrome://`、`chrome-extension://`、Chrome 应用商店等）不允许扩展注入脚本。请在普通网页（如 `https://` 开头的页面）上使用。

**Q：修改 Key 后数据丢失了？**

A：修改 Key 是「删除旧 Key → 创建新 Key」的非原子操作。正常情况下不会有问题，但如果在操作瞬间页面崩溃，理论上可能丢失。建议对重要数据先备份。

**Q：如何新增快速跳转链接？**

A：编辑 `src/popup.js` 中的 `jumpLinks` 数组，追加 `{ name: '显示名称', url: 'https://...' }` 对象即可。

**Q：如何修改白名单 Key 列表？**

A：编辑 `src/popup.js` 中的 `storageWhiteList` 数组，增删 Key 名称即可。白名单同时也决定了初始化时会写入哪些默认配置。
